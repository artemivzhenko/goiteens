Виведіть назву міста разом із фразою «is a wonderful place!».

Гілка andrii
[ADD] homework num2




Правила комітів
	•	[FIX] для виправлення помилок: зазвичай використовується у стабільній версії, але також доречно, якщо ви виправляєте нещодавню помилку у версії для розробки.
	•	[REF] для рефакторингу: коли функція значно переписується.
	•	[ADD] для додавання нових модулів.
	•	[REM] для видалення ресурсів: видалення непотрібного коду, видалення подань, видалення модулів тощо.

Додаткові
	•	[REV] для скасування комітів: якщо коміт викликає проблеми або не потрібний, його скасування здійснюється за допомогою цього тега.
	•	[MOV] для переміщення файлів: використовуйте git move і не змінюйте вміст переміщеного файлу, інакше Git може втратити зв’язок і історію файлу; також використовується при переміщенні коду з одного файлу в інший.
	•	[REL] для комітів випуску: нові основні або другорядні стабільні версії.
	•	[IMP] для покращень: більшість змін, внесених у версію для розробки, є поступовими покращеннями, які не відносяться до іншого тега.
	•	[MERGE] для комітів об’єднання: використовується для пересилання виправлень помилок, а також як основний коміт для функцій, що включають кілька окремих комітів.
